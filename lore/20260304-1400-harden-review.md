## Opportunity Details

### #1 - Body size limit (Quick/High)
- **What**: Change `usize::MAX` to a configurable limit (default 2MB) in `axum::body::to_bytes()`
- **Where**: `crates/mahalo-endpoint/src/endpoint.rs:121`
- **Why**: DoS vulnerability - clients can OOM the server with unlimited body

### #2 - PubSub subscribe panic (Quick/High)
- **What**: Return `Result` from `subscribe()` instead of `.expect()`. Handle send failure.
- **Where**: `crates/mahalo-pubsub/src/pubsub.rs:96-105`
- **Why**: Server crash if PubSub background task dies

### #3 - ChannelError traits (Quick/Medium)
- **What**: Derive/impl `std::fmt::Display` and `std::error::Error` for `ChannelError`
- **Where**: `crates/mahalo-channel/src/channel.rs:6-11`
- **Why**: Can't use with `Box<dyn Error>` or `?` operator in standard Rust error handling

### #4 - URL decode query params (Quick/Medium)
- **What**: Add percent-decoding to `parse_query_params()`. Use `percent_encoding` crate or manual decode.
- **Where**: `crates/mahalo-core/src/conn.rs:99-111`
- **Why**: `?name=hello%20world` currently gives literal `%20`

### #5 - Channel tests (Easy/High)
- **What**: Add unit tests for `ChannelRouter::find()`, `topic_matches()`, `ChannelSocket` assigns, `PhoenixMessage` serde, `Reply::ok/error`
- **Where**: `crates/mahalo-channel/src/socket.rs`, `crates/mahalo-channel/src/channel.rs`
- **Why**: 150+ lines of untested code in a critical module

### #6 - Plug tests (Easy/Medium)
- **What**: Add tests for `PlugFn` wrapper and `plug_fn()` helper
- **Where**: `crates/mahalo-core/src/plug.rs`
- **Why**: Core middleware abstraction has zero direct tests

### #7 - Standardize assigns (Easy/Medium)
- **What**: Use `AssignKey` trait pattern in `ChannelSocket` instead of raw `TypeId`
- **Where**: `crates/mahalo-channel/src/socket.rs:77-85`
- **Why**: Two different patterns for the same concept across the framework

### #8 - Decompose handle_websocket (Moderate/Medium)
- **What**: Extract `handle_join`, `handle_leave`, `handle_heartbeat`, `dispatch_event` helpers
- **Where**: `crates/mahalo-channel/src/socket.rs:132-282`
- **Why**: 150-line god function, high cyclomatic complexity, untestable

### #9 - Log send failures (Easy/Medium)
- **What**: Add `tracing::warn!` before `.ok()` / `let _ =` on send operations
- **Where**: `crates/mahalo-pubsub/src/pubsub.rs`, `crates/mahalo-channel/src/socket.rs`, `crates/mahalo-telemetry/src/telemetry.rs`
- **Why**: Silent failures make production debugging impossible

### #10 - Endpoint integration test (Moderate/High)
- **What**: Test that starts a MahaloEndpoint on a random port, sends HTTP requests, verifies Conn flows through plugs and returns correct responses
- **Where**: `crates/mahalo-endpoint/src/endpoint.rs` (test module)
- **Why**: The Axum bridge is the most critical path and has only unit tests for conversion functions
